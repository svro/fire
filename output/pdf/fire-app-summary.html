Fire

Static browser app

Who It Is For

Couples in Europe exploring FIRE scenarios with shared savings, retirement timing, pensions, inflation, and extra costs.

What It Is

Fire is a FIRE calculator for couples from Europe. It is a client-side web app that lets two people model portfolio growth and drawdown using ages, contributions, expenses, pensions, inflation, and returns.

What It Does

  • Models two-person timelines with separate current, retirement, pension, and end ages.
  • Charts portfolio value across three phases: two working, one working, and zero working.
  • Adjusts for annual return, inflation, current assets, annual contributions, and yearly spending.
  • Includes pension income for both people during retirement calculations.
  • Supports three configurable extra-cost periods with start and end years.
  • Can display projected values in nominal terms or today’s purchasing power.
  • Loads English, Dutch, French, and German text and remembers slider values in localStorage.

How To Run

  • Open the repo root containing index.html, script.js, style.css, and locales/.
  • Serve the folder over HTTP, because script.js loads locale files from /locales/{{lng}}.json and index.html pulls Chart.js and i18next from CDNs.
  • Open the served index.html page in a browser and adjust the sliders.
  • Exact start command or dev server: Not found in repo.

How It Works

UI shell: index.html defines a three-panel layout with sliders for person 1, shared settings, extras, person 2, plus a central canvas for the chart.
Presentation: style.css handles desktop and responsive mobile layout, slider styling, and card-like panels.
Client logic: script.js initializes i18next, restores saved inputs from localStorage, updates visible values, recalculates yearly portfolio data, and redraws the chart on every input change.
Data flow: user input -> DOM event listeners -> calculatePortfolioGrowth(...) -> phase-specific datasets -> Chart.js line chart and tooltips.
Translations: i18next fetches JSON files from locales/; browser language and stored preference choose the active locale, with English as fallback.
Backend/services: No repo-hosted backend, database, API, auth flow, or package-managed build system found in repo.
